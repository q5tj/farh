## كيف تختبر

### العميل: تذكرة دعم
1. سجّل دخول كعميل → **حسابي** → **الدعم الفني**
2. تبويب "تذكرة جديدة" → اكتب موضوع + وصف → **إرسال التذكرة**
3. ينقلك تلقائياً لتبويب "تذاكري" — تظهر التذكرة بحالة "جديدة"

### العميل: حول التطبيق
1. **حسابي** → **حول التطبيق**
2. تظهر ٣ بطاقات (فكرة، هدف، آلية) من `app_content`

### الأدمن: الرد على تذكرة
1. سجّل دخول كأدمن (يتطلب `users.role = 'admin'` تُضبط من Supabase Studio)
2. **حسابي** → **لوحة تحكم المالك** → **تذاكر الدعم**
3. تظهر التذكرة الجديدة → اضغط "الرد على التذكرة"
4. اكتب الرد، اختر الحالة (قيد المعالجة / مغلقة) → **حفظ الرد**
5. رجّع لحساب العميل → **الدعم الفني → تذاكري**: يظهر الرد في box بنفسجي.

### الأدمن: ترقية مستخدم
1. **لوحة تحكم → إدارة المستخدمين**
2. ابحث عن مستخدم → "ترقية إلى مزود خدمة"
3. يتحدث الدور في DB، البطاقة تتحدّث فوراً.
4. مع المستخدم: في **حسابي** يظهر "الانتقال لواجهة مزود الخدمة"، يقدر يدخل provider zone ويسوي onboarding.

### الأدمن: تعديل محتوى "حول التطبيق"
1. **لوحة تحكم → محتوى حول التطبيق**
2. عدّل النص بالعربية/الإنجليزية في أي بطاقة → **حفظ**
3. افتح حساب آخر **حسابي → حول التطبيق**: يظهر النص الجديد فوراً.

## نقاط أمان (RLS — كلها من v2)

| العملية | السياسة | من يقدر؟ |
|---|---|---|
| INSERT تذكرة دعم | `tickets_insert_self` | المستخدم يدرج تذكرته فقط |
| UPDATE تذكرة (رد) | `tickets_update_admin` | الأدمن فقط (`is_admin()`) |
| SELECT تذاكر | `tickets_select_visible` | المستخدم يرى تذاكره + الأدمن يرى الكل |
| UPDATE app_content | `app_content_write_admin` | الأدمن فقط |
| UPDATE users.role | `users_update_self` (مع check) | المستخدم لنفسه (لكن `admin` ممنوع — راجع v3). الأدمن من Supabase Studio أو RLS مخصصة |

> **ملاحظة**: ترقية الدور من واجهة الأدمن تعمل لأن سياسة `users_update_self` تسمح للمستخدم بتغيير دوره إلى `customer` أو `provider`. **لكن لا يمكن للواجهة جعل أحدهم admin** — هذا يتم فقط من Supabase Studio. السياسة الحالية تسمح للمستخدم بتغيير دوره لنفسه — لو تبي الأدمن يغيّر دور **مستخدم آخر**، تحتاج تضيف policy جديدة:
> ```sql
> create policy "users_update_admin" on public.users
>   for update using (public.is_admin())
>   with check (public.is_admin() and role in ('customer', 'provider'));
> ```

## استكشاف الأخطاء

| المشكلة | السبب |
|---|---|
| التذكرة لا تظهر بعد الإرسال | تحقق من `users.id` صحيح في الـ session. تحقق من سياسات `tickets_insert_self` و`tickets_select_visible`. |
| الأدمن لا يرى تذاكر المستخدمين الآخرين | تأكد إن `users.role = 'admin'` للمستخدم الحالي. الـ `is_admin()` helper يعتمد عليها. |
| محتوى "حول التطبيق" فاضي | تأكد من تنفيذ `migration_v2.sql` (يدرج الـ 3 entries: about_idea, about_goal, about_how). |
| ترقية المستخدم تعطي خطأ RLS | السياسة الحالية `users_update_self` تخلي المستخدم يغيّر دور **نفسه** فقط. لو تبي الأدمن يغيّر دور **مستخدم آخر**، أضف policy `users_update_admin` (انظر أعلاه). |
| dashboard stats لا تتحدّث بعد إجراء جديد | اسحب لتحديث (pull-to-refresh) — الـ stats تجلب عند تحميل الصفحة فقط (ليست realtime). |
